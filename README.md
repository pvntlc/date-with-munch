# 위드먼치 (date-with-munch)

먹고자와의 데이트를 사진·장소·한 일·일기로 기록하는 **설치형 웹앱(PWA)** 입니다.
무채색 미니멀 디자인.
모든 데이터는 **내 폰/브라우저(IndexedDB)에만** 저장되어 서버 없이 동작합니다.

## 기능
- 📔 **기록 타임라인** — 최신순 카드, 함께한 날/장소/사진 통계
- 🔍 **검색 & 필터** — 제목·장소·일기·태그 검색, 태그별 필터
- 🗓 **달력** — 날짜별로 기록을 한눈에(사진 썸네일 표시), 날짜 탭하면 그날 기록 보기
- 📍 **지역별 코스** — 다녀온 장소를 지역(구)별로 **자동 정리**, 각 장소를 **네이버 지도에서 바로 열기**
- 🔎 **네이버 장소 검색**(선택) — 장소 이름으로 검색해 고르면 주소·좌표·지역(구)이 자동 입력. 작은 프록시 서버 필요 → [server/README.md](server/README.md) 참고
- 💗 **기념일 · D-day** — 사귄 지 N일, 100일·주년 자동 카운트, 생일 등 나만의 기념일 D-day
- 💾 **백업 & 복원** — 모든 기록+사진을 파일 하나로 내보내고 복원 (기기 변경/재설치 대비)
- 📷 사진 여러 장 첨부(자동 리사이즈·압축), 🥰 기분, 🏷 태그, ✍️ 일기
- 🔒 영구 저장 요청으로 브라우저 자동 정리 방지

## 폰에 "앱처럼" 설치하기 (PWA)
빌드 도구 없이 폰에서 바로 설치됩니다.
1. PC에서 `npm run build` 후 `npm run preview` 또는 아무 정적 호스팅(Netlify/Vercel 등)에 `dist/` 배포
2. 폰 크롬/사파리에서 그 주소 접속 → 메뉴 → **"홈 화면에 추가"**
3. 홈 화면 아이콘으로 전체화면 앱처럼 실행됩니다. (오프라인에서도 열림)

> PWA는 네이티브 APK와 달리 **다시 설치해도 데이터가 보존**됩니다(브라우저 저장소 유지).
> 그래도 기기 변경·브라우저 데이터 삭제에 대비해 **더보기 → 백업**을 주기적으로 권장합니다.

## 실행 방법
```bash
npm install            # 최초 1회
npm run dev            # 개발 서버 (http://localhost:5173)
npm run build          # 배포용 빌드 (dist/) — PWA 매니페스트/서비스워커 포함
npm run preview        # 빌드 결과 미리보기 (http://localhost:4173)
node scripts/gen-icons.mjs   # 앱 아이콘 재생성(아이콘 디자인 바꿀 때만)
```

## 기술
- React 18 + Vite
- PWA: `vite-plugin-pwa` (서비스워커·매니페스트·오프라인 캐시)
- 저장: IndexedDB (`idb`) — 사진은 Blob, 백업 시 base64(JSON)로 직렬화

## 데이터 관리 주의
- 데이터는 이 기기/브라우저에만 저장됩니다. 다른 기기에서는 백업 파일로 옮겨야 보여요.
- 브라우저 데이터를 완전히 지우면 기록도 사라집니다 → **백업 파일을 클라우드/메일에 보관 권장**.
